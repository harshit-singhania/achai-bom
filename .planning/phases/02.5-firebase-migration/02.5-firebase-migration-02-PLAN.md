# Plan 02: Rewrite Repositories for Firestore

## Goal
Rewrite all 5 repository files to use `get_firestore_client()` instead of `get_session()`, preserving the existing dict-return contract.

## must_haves
- All 5 repositories use Firestore client instead of SQLAlchemy sessions
- Collection names: `projects`, `floorplans`, `materials_pricing`, `generated_boms`, `async_jobs`
- All `create_*()` return `str` (was `int`)
- All `get_*_by_id(id: str)` accept `str` (was `int`)
- Dict return contract preserved exactly
- Non-blocking persistence pattern preserved

<tasks>

<task id="1" title="Rewrite project_repository.py">
Replace SQLAlchemy session usage with Firestore:
- `create_project(...)` → `db.collection("projects").document()` with auto-ID, returns `str`
- `get_project_by_id(id: str)` → `db.collection("projects").document(id).get()`
- `update_project(id: str, ...)` → `db.collection("projects").document(id).update({...})`
- Preserve dict return format with all existing fields
- Import `get_firestore_client` from `app.core.firebase`
</task>

<task id="2" title="Rewrite floorplan_repository.py">
Replace SQLAlchemy session usage with Firestore:
- `create_floorplan(...)` → `db.collection("floorplans").document()`, returns `str`
- `get_floorplan_by_id(id: str)` → document get
- `get_floorplans_by_project(project_id: str)` → `.where("project_id", "==", project_id)` query
- `update_floorplan(id: str, ...)` → document update
- Preserve dict return format
</task>

<task id="3" title="Rewrite bom_repository.py">
Replace SQLAlchemy session usage with Firestore:
- `create_bom(...)` → `db.collection("generated_boms").document()`, returns `str`
- `get_bom_by_id(id: str)` → document get
- `list_boms_by_project(project_id: str)` → 2-step query: get floorplan IDs for project, then Firestore `in` query on `floorplan_id` (max 30 values per `in` clause)
- `update_bom(id: str, ...)` → document update
- Preserve dict return format
</task>

<task id="4" title="Rewrite job_repository.py">
Replace SQLAlchemy session usage with Firestore:
- `create_job(...)` → `db.collection("async_jobs").document()`, returns `str`
- `get_job_by_id(job_id: str)` → document get
- `mark_job_running(job_id: str)` → `document.update({"status": "running", "started_at": ...})`
- `mark_job_complete(job_id: str, result: dict)` → document update with result and completed_at
- `mark_job_failed(job_id: str, error: str)` → document update with error and completed_at
- `list_jobs_by_floorplan(floorplan_id: str)` → `.where("floorplan_id", "==", id).order_by("created_at", direction=DESCENDING)`
- Job lifecycle: single-document updates are atomic in Firestore
</task>

<task id="5" title="Rewrite materials_repository.py">
Replace SQLAlchemy session usage with Firestore:
- `get_all_materials()` → `db.collection("materials_pricing").stream()`, return list of dicts
- `get_material_by_id(id: str)` → document get
- `get_materials_by_category(category: str)` → `.where("category", "==", category)` query
- `create_material(...)` → document set, returns `str`
- `update_material(id: str, ...)` → document update
- Firestore Timestamps → `.isoformat()` in serialization helpers
</task>

</tasks>
