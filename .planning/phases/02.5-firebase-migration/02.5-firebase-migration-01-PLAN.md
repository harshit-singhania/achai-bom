# Plan 01: Firebase Core Infrastructure + Models

## Goal
Replace Supabase/SQLAlchemy infrastructure with Firebase Admin SDK and convert SQLModel ORM classes to plain Pydantic BaseModel.

## must_haves
- Firebase Admin SDK initialized via service account JSON
- Firestore client singleton with lazy init and reset capability
- All 5 model classes converted from SQLModel to plain Pydantic BaseModel
- ID fields changed from `int` to `str` across all models
- Supabase config vars removed, Firebase config vars added
- `app/core/database.py` deleted (no consumers after repo rewrite in Plan 02)

<tasks>

<task id="1" title="Update app/core/config.py">
Remove Supabase settings, add Firebase settings:
- Remove: `SUPABASE_PROJECT_URL: str`, `SUPABASE_PUBLISHABLE_KEY: str`, `SUPABASE_PASSWORD: str`, `DATABASE_URL: str`
- Add:
  ```python
  # Firebase Settings
  FIREBASE_CREDENTIALS_PATH: str = ""
  FIREBASE_PROJECT_ID: str = ""
  FIRESTORE_EMULATOR_HOST: str = ""
  ```
</task>

<task id="2" title="Create app/core/firebase.py" depends_on="1">
Lazy-initialized Firestore client singleton (mirrors existing database.py pattern):
- `get_firestore_client() -> Client` — init Firebase Admin SDK on first call, return Firestore client
- `reset_firestore_client() -> None` — for test isolation (clears global `_app` and `_db`)
- Auth: service account JSON via `FIREBASE_CREDENTIALS_PATH`, or Application Default Credentials
- Global `_app` and `_db` with lazy init
- Import `firebase_admin` and `google.cloud.firestore`
</task>

<task id="3" title="Delete app/core/database.py" depends_on="2">
Delete the file entirely. No consumers outside repositories (rewritten in Plan 02) and tests (rewritten in Plan 04). The new `firebase.py` replaces it.
</task>

<task id="4" title="Convert app/models/database.py to Pydantic" depends_on="2">
Convert 5 SQLModel ORM classes to plain Pydantic `BaseModel`:
- Remove: `SQLModel`, `Relationship`, `sa_type=JSON`, `foreign_key`, `__tablename__`, `table=True`
- Keep: `Field(description="...")`, `default_factory` patterns
- **ID change:** `id: Optional[int]` → `id: str | None = None`
- **FK change:** all FK fields (`project_id`, `floorplan_id`) from `int` → `str | None`
- Remove relationship fields (`floorplans`, `project`, `boms`)
- Classes to convert: `Project`, `Floorplan`, `MaterialsPricing`, `GeneratedBOM`, `AsyncJob`
</task>

<task id="5" title="Update requirements.in + regenerate lock" depends_on="2">
- Remove: `sqlmodel>=0.0.16`, `supabase>=2.3.0`
- Add: `firebase-admin>=6.0.0`
- Run: `pip-compile requirements.in --output-file requirements-lock.txt --strip-extras`
</task>

<task id="6" title="Update .env.example" depends_on="1">
- Remove Supabase vars (`SUPABASE_PROJECT_URL`, `SUPABASE_PUBLISHABLE_KEY`, `SUPABASE_PASSWORD`, `DATABASE_URL`)
- Add Firebase vars with comments:
  ```
  # Firebase
  FIREBASE_CREDENTIALS_PATH=path/to/serviceAccountKey.json
  FIREBASE_PROJECT_ID=your-project-id
  FIRESTORE_EMULATOR_HOST=localhost:8080  # for local dev/testing
  ```
</task>

</tasks>
