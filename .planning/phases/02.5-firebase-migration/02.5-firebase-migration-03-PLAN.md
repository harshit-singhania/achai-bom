# Plan 03: Type Propagation (int→str) + Seed Script

## Goal
Propagate the `int` → `str` ID type change through all service, worker, and API layers. Create Firebase seed script to replace Supabase schema setup.

## must_haves
- All ID parameters across services, workers, and routes accept `str` (not `int`)
- Route params use string patterns (no `<int:...>` converters)
- Seed script populates Firestore with same 45 materials + sample data
- Old Supabase schema script deleted

<tasks>

<task id="1" title="Update app/api/routes.py">
- Change route params: `<int:pdf_id>` → `<pdf_id>`, `<int:job_id>` → `<job_id>`
- Update parameter type annotations: `int` → `str` for all ID params
- Remove any `int()` casts on ID values from request data
</task>

<task id="2" title="Update app/services/ingest_service.py">
- `IngestJobResult.job_id: int` → `str`
- `IngestJobResult.floorplan_id: int | None` → `str | None`
- Remove `int(project_id_raw)` cast, replace with non-empty string validation
- `floorplan_id: int | None` → `str | None` in function signatures
</task>

<task id="3" title="Update app/services/generate_service.py">
- `GenerateJobResult.job_id: int` → `str`
- `floorplan_id: int | None` → `str | None` in function params
- `isinstance(floorplan_id, int)` → `isinstance(floorplan_id, str)` in validation checks
</task>

<task id="4" title="Update app/services/status_service.py">
- `def get_floorplan_status(pdf_id: int)` → `(pdf_id: str)`
- `def get_job_status(job_id: int)` → `(job_id: str)`
</task>

<task id="5" title="Update app/workers/job_runner.py">
- `def run_job(job_id: int)` → `(job_id: str)`
- `floorplan_id: Optional[int]` → `Optional[str]` in all relevant locations
</task>

<task id="6" title="Update app/workers/queue_worker.py">
- `db_job_id: int` → `db_job_id: str` in all relevant locations
</task>

<task id="7" title="Create scripts/seed_firebase.py" depends_on="1,2,3,4,5,6">
New seed script replacing `scripts/setup_supabase_schema.py`:
- Same 45 materials across 10 categories with Indian market pricing
- Same sample project/floorplan/BOM for testing
- Uses `get_firestore_client()` from `app.core.firebase`
- CLI entry point: `python scripts/seed_firebase.py`
- Idempotent: check if materials collection already has data before seeding
</task>

<task id="8" title="Delete scripts/setup_supabase_schema.py" depends_on="7">
Delete the old Supabase schema setup script entirely.
</task>

</tasks>
